## 核心概念

### 任务管理原则（Manus 风格）

1. **用文件做记忆** - Context Window = RAM（易失），Filesystem = Disk（持久）
2. **先计划后执行** - 任何复杂任务先写 plan
3. **每 2 次操作保存发现** - 防止信息丢失
4. **决策前读计划** - 保持目标在注意力窗口
5. **行动后更新** - 标记完成、记录错误

### 定时任务限制原则

1. **临时任务必设限** - 避免遗忘导致消息轰炸
2. **达到限制自动停** - 不删除，保留记录可追溯
3. **定期检查状态** - 运行 check 命令清理
4. **修改前先备份** - 防止配置丢失

## Cron 表达式速查

| 表达式 | 含义 | 使用场景 |
|--------|------|----------|
| `30 7 * * *` | 每天 7:30 | 起床提醒 |
| `30 6 * * *` | 每天 6:30 | 早起提醒 |
| `*/5 * * * *` | 每 5 分钟 | 高频测试（慎用） |
| `0 * * * *` | 每小时整点 | 整点提醒 |
| `0 0 * * *` | 每天午夜 | 每日清理 |
| `0 0 * * 1` | 每周一午夜 | 周任务 |
| `30 6,18 * * *` | 每天 6:30 和 18:30 | 早晚提醒 |
| `0 9 * * 1-5` | 工作日 9:00 | 上班打卡提醒 |

### Cron 格式说明

```
* * * * *
│ │ │ │ │
│ │ │ │ └─ 星期几 (0-7, 0 和 7 都是周日)
│ │ │ └─── 月份 (1-12)
│ │ └───── 日期 (1-31)
│ └─────── 小时 (0-23)
└───────── 分钟 (0-59)
```

特殊字符：
- `*` - 每个值（每分钟、每小时等）
- `,` - 分隔多个值（如 `6,18` 表示 6 点和 18 点）
- `-` - 范围（如 `1-5` 表示周一到周五）
- `/` - 步长（如 `*/5` 表示每 5 分钟）

## 限制配置示例

### 只限制次数

```json
"limits": {
  "maxRuns": 10,
  "currentRuns": 0
}
```

### 只限制时间

```json
"limits": {
  "expireAt": "2026-03-09T18:00:00+08:00"
}
```

### 同时限制

```json
"limits": {
  "maxRuns": 10,
  "currentRuns": 0,
  "expireAt": "2026-03-09T18:00:00+08:00"
}
```

### 无限制（默认）

```json
"limits": null
// 或不设置 limits 字段
```
