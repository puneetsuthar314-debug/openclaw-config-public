# Task & Cron Manager - 任务与定时任务管理技能

*版本：2.0 | 创建日期：2026-03-08 | 更新日期：2026-03-08*

---

## 📖 技能描述

本技能提供完整的任务管理和定时任务系统：

1. **大型任务规划** - 为每个复杂任务创建独立的 `todo_XXX.md` 文件
2. **定时任务系统** - 支持执行次数限制的 cron 任务
3. **自动化脚本** - 任务检查、归档、报告生成

---

## 🎯 核心概念

### 📋 todo_XXX.md - 大型任务规划文件

**定义：** 当你遇到**大型/复杂任务**时，临时创建的规划文件，帮助你：
- 确认方向，不跑偏
- 拆解步骤，一步步完成
- 记录进度，随时回顾

**特点：**
| 特征 | 说明 |
|------|------|
| **临时性** | 任务完成后归档或删除 |
| **独立性** | 每个大型任务一个独立文件 |
| **命名灵活** | `todo_XXX.md` / `todo_001.md` / `任务名称.md` |
| **不共享** | 不同任务之间不共用 |
| **轻量级** | 快速创建，快速使用，快速清理 |

**何时创建：**
- ✅ 需要多步骤完成的复杂任务
- ✅ 需要跨会话持续跟进的项目
- ✅ 容易跑偏或忘记目标的大型工作
- ❌ 简单任务（直接做，不需要规划文件）
- ❌ 日常小任务（不需要专门文件）

---

### 📁 tasks/ - 长期任务档案（可选）

用于归档已完成的 todo 文件和长期项目追踪，**不是必须使用的**。

## 📁 文件结构

```
/root/.openclaw/workspace/
├── todo_XXX.md                    # 当前活跃的大型任务规划（可能有多个）
├── todo_001-任务名称.md           # 按序号或任务名区分
├── tasks/
│   ├── archive/                   # 已完成的 todo 文件归档
│   │   └── global-todo-2026-03-08.md
│   └── template.md                # 任务规划模板
└── scripts/
    ├── cron-manager.py            # 定时任务管理
    └── task-utils.py              # 任务工具（归档、清理）

/root/.openclaw/cron/
├── jobs.json                      # 定时任务配置
└── jobs-template.json             # 任务模板
```

## 🔧 使用流程

### 遇到大型任务时

```
1. 判断是否需要规划文件
   → 需要多步骤？→ 是 → 创建 todo_XXX.md
   → 简单任务？→ 否 → 直接执行

2. 创建规划文件
   → 使用模板：tasks/template.md
   → 命名：todo_任务名称.md 或 todo_001.md

3. 执行过程中
   → 每完成一步，在文件中打勾
   → 有新发现，记录到文件中
   → 方向偏离，回顾文件中的目标

4. 任务完成后
   → 移动到 tasks/archive/ 归档
   → 或直接删除（如果不重要）
```

## ⏰ 定时任务管理

### 创建带限制的定时任务

```json
{
  "id": "unique-id",
  "name": "任务名称",
  "enabled": true,
  "schedule": {
    "kind": "cron",
    "expr": "*/5 * * * *",
    "tz": "Asia/Shanghai"
  },
  "limits": {
    "maxRuns": 10,
    "currentRuns": 0,
    "expireAt": "2026-03-09T00:00:00+08:00"
  },
  "payload": {
    "kind": "agentTurn",
    "message": "任务内容"
  },
  "delivery": {
    "mode": "announce",
    "channel": "clawdbot-dingtalk",
    "to": "target"
  }
}
```

### 管理命令

```bash
# 列出所有定时任务
python /root/.openclaw/workspace/scripts/cron-manager.py list

# 检查并禁用达到限制的任务
python /root/.openclaw/workspace/scripts/cron-manager.py check

# 手动增加执行计数
python /root/.openclaw/workspace/scripts/cron-manager.py increment <task-id>
```

## ⚠️ 最佳实践

### todo_XXX.md 使用原则

1. **只在需要时创建** - 不要为每个小任务都创建文件
2. **快速创建快速用** - 5 分钟内完成规划，立刻开始执行
3. **保持轻量** - 不要过度设计，够用就行
4. **完成后清理** - 归档或删除，不要堆积
5. **命名要清晰** - 一眼能看出是什么任务的规划

### 定时任务原则

1. **临时任务必设限** - 避免遗忘导致消息轰炸
2. **达到限制自动停** - 不删除，保留记录可追溯
3. **定期检查状态** - 运行 check 命令清理
4. **修改前先备份** - 防止配置丢失

## 🚫 常见误区

### ❌ 错误做法

| 误区 | 问题 | 正确做法 |
|------|------|----------|
| 全局唯一 todo.md | 所有任务混在一起，难以管理 | 每个大型任务独立 todo_XXX.md |
| 小任务也创建 todo | 文件泛滥，维护成本高 | 简单任务直接做 |
| 完成后不归档 | 根目录堆积大量文件 | 移动到 archive/ 或删除 |
| 过度规划 | 花 1 小时规划，5 分钟执行 | 5 分钟规划，立刻执行 |
| 所有进程共用 | 不同任务互相干扰 | 独立文件，互不干扰 |
