# Wide Search - 自主深度研究技能

**类似 Manus 的多轮迭代搜索能力**

---

## 🎯 功能特点

- ✅ 多轮迭代搜索（自主决定搜索关键词）
- ✅ 跨多个信息源验证
- ✅ 自动解析网页内容
- ✅ 综合整理成报告
- ✅ 支持并行搜索（多 Agent）
- ✅ **免费**（使用现有 web_search + browser 工具）

---

## 📋 使用方式

### 方式 1：极简任务卡（推荐）

创建 `todo_研究任务.md`：

```markdown
## 🎯 方向
研究 Python 异步编程最佳实践

## ✅ 预期效果
生成一份 10 页+ 的技术报告，包含代码示例和性能对比

## ⚠️ 限制条件
- 搜索至少 15 个来源
- 包含 2024-2026 年的最新资料
- 要有实际代码示例
```

### 方式 2：直接指令

> "Claw，帮我深度研究 [主题]，生成详细报告"

## 🤖 自主研究流程

### 阶段 1：规划（5 分钟）
1. 理解研究主题
2. 拆解为子问题
3. 制定搜索策略
4. 确定信息来源

### 阶段 2：搜索（10-30 分钟）
1. 第一轮搜索（5-10 个关键词）
2. 阅读搜索结果摘要
3. 决定深入方向
4. 第二轮搜索（细化关键词）
5. 解析关键网页内容
6. 提取数据和引用

### 阶段 3：综合（10-20 分钟）
1. 整理所有信息
2. 交叉验证数据
3. 识别矛盾/不一致
4. 形成结论

### 阶段 4：报告（10-15 分钟）
1. 生成结构化报告
2. 添加引用来源
3. 编写执行摘要
4. 输出最终文档

## 📊 搜索策略

### 多轮迭代
```
Round 1: [主题] 概述
  ↓
Round 2: [主题] + 技术细节/最佳实践
  ↓
Round 3: [主题] + 性能对比/案例分析
  ↓
Round 4: [主题] + 2024 2025 2026 最新进展
```

### 多源验证
- 官方文档
- 技术博客
- GitHub 项目
- Stack Overflow
- 学术论文（如适用）
- 行业报告

### 并行搜索（可选）
对于大型研究任务：
```
主 Agent（协调）
├── Agent 1: 技术实现搜索
├── Agent 2: 性能对比搜索
├── Agent 3: 最佳实践搜索
└── Agent 4: 案例研究搜索
```

## 📁 输出结构

### 标准报告格式

```markdown
# [研究主题] 深度研究报告

## 执行摘要
[300 字概述核心发现]

## 目录
1. 背景与概述
2. 技术细节
3. 最佳实践
4. 性能对比
5. 案例分析

[所有来源列表，带链接]
```

### 研究日志
同时生成 `research-日志.md` 记录：
- 每轮搜索关键词
- 访问的网页
- 提取的关键信息
- 决策过程

## ⚙️ 配置选项

### 在任务文件中指定

```markdown
## 研究配置
- 搜索轮数：3-5 轮
- 最少来源：15 个
- 时间范围：2024-2026
- 语言：中文 + 英文
- 输出格式：Markdown + JSON
- 预计时间：30-60 分钟
```

### 自主决策（不指定时）

- 搜索轮数：根据主题复杂度自主决定
- 来源数量：至少 10 个高质量来源
- 时间范围：优先最新 2 年
- 语言：自动选择（中文优先，英文补充）

## 🛠️ 技术实现

### 使用的工具
- `web_search` - 多轮搜索
- `aliyun_web_parser` - 网页内容解析
- `browser` - 深度浏览（如需要）
- `sessions_spawn` - 并行搜索（大型任务）

### 搜索质量保障
1. **来源评估** - 优先官方文档、权威网站
2. **交叉验证** - 关键数据至少 2 个来源确认
3. **时效性** - 优先最新内容
4. **多样性** - 避免单一信息源偏见

## 🚫 限制说明

### 不能做的
- ❌ 访问付费墙后的内容
- ❌ 需要登录才能访问的网站
- ❌ 实时数据（股票、天气等）
- ❌ 私人/内部文档

### 替代方案
- 使用公开摘要
- 寻找替代来源
- 请求用户提供访问权限
