## 🎯 产品定位

**为 IP 创作打造的 AI 写小说系统**

不是卖订阅费的工具，而是**我们自己的核心竞争力**。

### 核心价值
- 高效产出优质内容
- 保障长文本连贯性
- 沉淀创作知识和风格
- 服务多部作品复用

---

## 📋 核心功能模块

### P0 - 核心功能（必须实现）

#### 1. 大纲生成系统
**功能：**
- 总纲生成（世界观/主线/结局）
- 分卷大纲（每卷 10-20 章）
- 章节大纲（单章要点）
- 支持手动修改和迭代

**输入：**
- 题材类型（玄幻/言情/都市/悬疑）
- 核心创意（1-2 句话）
- 字数预期（总字数/单章字数）

**输出：**
- 完整大纲文档（Markdown 格式）
- 可编辑、可迭代

**技术实现：**
- 阿里云百炼 API（qwen3.5-plus）
- 结构化 Prompt 模板
- 分步生成（总纲→分卷→章节）

---

#### 2. 角色管理系统
**功能：**
- 角色档案（姓名/年龄/外貌/性格/背景）
- 角色关系图（亲属/朋友/敌人/爱人）
- 成长弧线（初始状态→关键事件→最终状态）
- 角色一致性检查

**输入：**
- 角色基本信息
- 角色关系描述
- 成长节点

**输出：**
- 角色档案库（JSON/Markdown）
- 关系图谱（文本描述）
- 一致性检查报告

**技术实现：**
- 结构化数据存储（JSON）
- 角色一致性 Prompt 检查
- 可视化（可选：mermaid 图谱）

---

#### 3. 章节创作系统
**功能：**
- 根据大纲生成章节初稿
- 支持续写（给定前文，生成后续）
- 支持润色（优化文笔/节奏/对话）
- 支持风格化（模仿特定文风）

**输入：**
- 章节大纲
- 前文内容（上下文）
- 风格要求（可选）

**输出：**
- 章节初稿（4000-6000 字）
- 可编辑、可迭代

**技术实现：**
- 阿里云百炼 API
- 上下文窗口管理（长文本记忆）
- 风格化 Prompt 模板

---

#### 4. 上下文记忆系统
**功能：**
- 长文本连贯性保障
- 关键信息记忆（人名/地名/设定）
- 剧情线索追踪
- 自动提醒矛盾点

**输入：**
- 全部已写内容
- 关键信息提取

**输出：**
- 记忆库（关键信息索引）
- 矛盾点报告

**技术实现：**
- 向量数据库（可选）
- 关键信息提取 Prompt
- 矛盾检测规则

---

### P1 - 辅助功能（第二优先级）

#### 5. 世界观管理系统
**功能：**
- 世界观设定（历史/地理/种族/势力）
- 规则体系（魔法/科技/修炼体系）
- 时间线管理（大事年表）

---

#### 6. 质量检查系统
**功能：**
- 逻辑一致性检查
- 人设一致性检查
- 剧情连贯性检查
- AI 痕迹检测与优化

---

#### 7. 灵感库系统
**功能：**
- 素材收集（桥段/金句/场景）
- 爆款分析（热门作品拆解）
- 灵感推荐（基于当前剧情）

---

### P2 - 数据功能（第三优先级）

#### 8. 数据分析系统
**功能：**
- 读者反馈分析
- 平台数据追踪（阅读/收藏/评论）
- 收益统计
- A/B 测试（不同章节/风格）

---

## 🏗️ 技术架构

### 前端（可选）
- 简单网页界面（HTML/CSS/JS）
- 或命令行界面（Python）
- 或直接用 Markdown 文件 + 脚本

### 后端
- Python 脚本
- 阿里云百炼 API 调用
- 本地文件存储（Markdown/JSON）

### 数据存储
- 项目文件夹结构
- Markdown 文件（内容）
- JSON 文件（结构化数据）

## ✅ 验收标准

### 功能验收
- [ ] 能生成完整大纲（总纲 + 分卷 + 章节）
- [ ] 能管理角色档案和关系
- [ ] 能生成 4000+ 字章节，连贯性好
- [ ] 能记忆关键信息，不矛盾

### 质量验收
- [ ] 生成内容有人味，不机械
- [ ] 逻辑自洽，人设不崩
- [ ] 风格一致，可识别

### 效率验收
- [ ] 生成大纲 <5 分钟
- [ ] 生成章节 <2 分钟
- [ ] 日常使用流畅
