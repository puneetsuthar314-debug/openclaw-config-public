# 快速启动项目指南

## 🚀 最简用法（推荐）

### 你只需要做 3 件事

**1. 创建任务文件**
```bash
# 复制模板
cp /root/.openclaw/workspace/templates/极简任务卡.md \
   /root/.openclaw/workspace/todo_任务名.md
```

**2. 填写 3 项内容**
```markdown
## 🎯 方向
做一个课表提醒机器人

## ✅ 预期效果
每天早上 8 点自动推送当天课程到钉钉

## ⚠️ 限制条件（可选）
只能用 Python，不能用付费 API
```

**3. 告诉我开始执行**
> "Claw，todo_课表机器人.md 已创建，开始执行"

## 📋 完整流程示例

### 你说
> "我想做一个自动备份脚本，每天备份工作区到阿里云 OSS"

### 我会
1. ✅ 创建 `todo_自动备份.md`（如未创建）
2. ✅ 调研阿里云 OSS Python SDK
3. ✅ 设计备份方案（全量/增量、压缩、加密）
4. ✅ 编写备份脚本
5. ✅ 编写测试用例
6. ✅ 执行测试（模拟备份）
7. ✅ 配置定时任务
8. ✅ 编写使用文档
9. ✅ 生成测试报告
10. ✅ 归档项目

### 过程中
- 每日更新 `projects/自动备份/progress.md`
- 遇到问题自主搜索解决
- 完成后通知你验收

## 💡 高级用法

### 并行子 Agent（大型项目）
在任务文件中添加：
```markdown
## 执行模式
使用 3 个子 Agent 并行：
- Agent 1: 核心开发
- Agent 2: 测试
- Agent 3: 文档
```

### 指定技术栈
```markdown
## 限制条件
- 使用 Python 3.10+
- 使用 FastAPI 框架
- 使用 SQLite 数据库
- 不能用 Docker
```

### 指定交付时间
```markdown
## 限制条件
- 本周五前完成
- 每天汇报进度
```

## 🎯 最佳实践

### ✅ 好的任务描述
- "做一个网页爬虫，爬取 XX 网站的产品信息，保存为 Excel"
- "写个脚本自动整理我的照片，按日期分类"
- "开发一个待办事项 API，支持增删改查"

### ❌ 避免模糊描述
- "帮我做个很厉害的东西"（太模糊）
- "优化一下系统"（不明确）
- "让代码更好"（无衡量标准）

### 💡 提示
- **方向**：清晰描述要做什么
- **预期效果**：可衡量的完成标准
- **限制条件**：有就写，没有就不填
