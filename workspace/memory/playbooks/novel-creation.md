# 小说创作剧本

> 最后更新：2026-03-12

## 适用场景
用户要求创作中长篇小说，需要全流程自动化完成，包含质量审查机制。

## 推荐模型
- **设定阶段**: qwen3-max-2026-01-23（复杂推理、世界观构建）
- **创作阶段**: MiniMax-M2.5（长文写作、创意内容）
- **审查阶段**: qwen3-max-2026-01-23（逻辑检查、一致性验证）

## 标准流程

### 第一阶段：设定构建（1-2 轮）
1. **市场调研** - web_search 查询同类作品趋势
2. **核心设定** - 类型、主题、受众、篇幅
3. **世界观** - 背景、规则、势力分布
4. **人物群像** - 主角、反派、配角详细人设

### 第二阶段：架构设计（1-2 轮）
1. **主线情节** - 三幕式结构（起因→发展→转折→高潮→结局）
2. **支线规划** - 感情线、成长线、辅线
3. **章节大纲** - 每章核心事件、爽点、冲突点（20-50 章）

### 第三阶段：章节创作（多轮）
1. **分章撰写** - 每章 2000-4000 字，使用 MiniMax-M2.5
2. **质量审查** - 逻辑检查、上下文一致性、文笔优化
3. **迭代修改** - 根据审查反馈调整
4. **用户汇报** - 每 5 章汇报一次进度

### 第四阶段：统稿精修（1-2 轮）
1. **全文通读** - 检查连贯性、节奏感
2. **细节打磨** - 对话优化、描写增强
3. **最终校对** - 错别字、语病、逻辑漏洞

## 工具链
- web_search → 市场调研、趋势分析
- MiniMax-M2.5 → 长文创作
- qwen3-max-2026-01-23 → 逻辑审查、设定构建
- filesystem → 保存章节文件、设定文档

## 关键决策点
- **类型选择**: 需结合市场趋势和用户偏好（玄幻/都市/科幻/言情/悬疑）
- **核心冲突**: 决定故事吸引力（复仇/成长/爱情/冒险/权谋）
- **结局设计**: 影响整体评价（HE/BE/开放式）

## 质量标准
- 情节连贯，无逻辑漏洞
- 人物行为符合人设，有成长弧光
- 节奏合理，爽点分布均匀
- 文笔流畅，描写生动
- 上下文一致，无矛盾

## 踩坑记录
- 避免直接生成全文不审查（容易逻辑崩坏）
- 避免人物设定过于扁平（需要前史、动机、缺陷）
- 避免节奏拖沓（每章需有核心事件和冲突）
