# 2026-03-08 - 钉钉群聊定时消息限制诊断

## 事件

用户要求每隔 1 分钟在群里@陈玺翔 讲笑话，共 10 次。

## 问题发现

定时任务创建成功，但消息未送达钉钉群。查看 cron 运行记录显示：
```
"delivered": false
"deliveryStatus": "not-delivered"
"deliveryError": "cron announce delivery failed"
```

## 诊断过程

1. **检查 cron 任务状态** - 任务已注册，运行成功，但投递失败
2. **测试 message 工具** - 尝试多种目标格式：
   - `dingtalk:group:cid...` ❌
   - `cid...` ❌
   - `group:cid...` ❌
   - `@cid...` ❌
   - `dingtalk:manager20` ❌
   - `manager20` ✅ (私聊工作)

3. **根本原因** - 钉钉渠道的 `message` 工具和 cron announce 机制只支持私聊目标格式（senderStaffId），群聊目标格式不被识别。

## 解决方案

### 方案 A：私聊提醒（立即可用）
使用 senderStaffId 发送私聊消息。

### 方案 B：钉钉群机器人 webhook
通过 HTTP POST 直接发送到群聊：
```bash
curl 'https://oapi.dingtalk.com/robot/send?access_token=YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{"msgtype":"text","text":{"content":"@某人 提醒内容"}}'
```

### 方案 C：本地定时脚本
编写 Node.js/Python 脚本定时调用 webhook。

已创建脚本：`/root/.openclaw/workspace/scripts/joke-timer.js`

## 经验教训

1. **钉钉渠道限制** - 群聊定时消息目前不支持，需在 skill 文档中明确说明
2. **测试目标格式** - 遇到投递失败时，应系统测试所有可能的目标格式
3. **webhook 备用方案** - 钉钉群机器人 webhook 是可靠的替代方案
