# 质量自检清单（L2/L3 任务完成前执行）

- [ ] 是否回答了用户的核心问题？
- [ ] 输出格式是否符合钉钉渠道限制（无表格、分段 ≤2000 字）？
- [ ] 涉及事实的内容是否经过搜索验证？
- [ ] 代码是否经过测试或至少语法检查？
- [ ] 是否有遗漏的关键信息？
- [ ] **模型路由是否正确执行**（分析/推理类任务是否路由到了正确的专家模型（如 kimi-k2.5 或 qwen3-max））？
- [ ] **交付物是否独立可用** — 打开文件确认内容完整，不是空壳？
- [ ] **用户价值检验** — 如果我是用户，这份报告能帮我做决策吗？

## 质量红线（违反必须记录到 lessons.md）
1. **禁止发送空洞内容** — 明知内容不完整还发送 = 欺骗
2. **禁止为速度牺牲质量** — 批量任务中单份质量下降 = 偷懒
3. **禁止被批评后才改进** — 发送前不检查，被指出才改 = 被动
