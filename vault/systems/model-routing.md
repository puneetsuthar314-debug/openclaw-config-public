# 智能模型路由

## 核心架构：Qwen3.5-Plus 指挥 + 专家团队干活

> **丫丫的大脑是 Qwen3.5-Plus，它负责理解需求、拆解任务、做出决策。具体的执行工作交给专家模型完成。Qwen3.5-Plus 出错时自动降级到 qwen3-max 或 kimi-k2.5。**

## 模型能力矩阵

| 模型 | 别名 | 角色 | 最佳场景 | 上下文窗口 | 推理能力 |
|------|------|------|----------|-----------|---------|
| **qwen3.5-plus** | **默认** | **主控大脑** | **任务理解、需求分析、决策、任务分发、日常对话** | 1M | 否 |
| qwen3-coder-plus | coder | 编码专家 | 写代码、代码审查、技术架构、调试 | 1M | 否 |
| kimi-k2.5 | kimi | 研究专家 + 视觉专家 | 深度研究、图片理解、视觉分析、多模态任务 | 262K | 是 |
| qwen3-max-2026-01-23 | max | 推理专家 | 复杂推理、数学、逻辑分析 | 262K | 是 |
| qwen3-coder-next | coder-next | 压缩专家 | 代码生成（更新版本，上下文较小） | 262K | 否 |
| MiniMax-M2.5 | minimax | 写作专家 | 长文写作、创意内容 | 204K | 是 |
| glm-4.7 | glm4 | 后台守护 | 心跳检查、低成本后台任务 | 169K | 是 |

## 自动切换规则（硬性规则）

> **铁律：Qwen3.5-Plus 是默认主控模型。只有在以下明确场景中，才切换到专家模型执行具体工作。执行完毕后必须切回 Qwen3.5-Plus。**

收到用户消息后，Qwen3.5-Plus 先理解需求，然后根据以下规则判断是否需要切换到专家模型：

**强制切换（静默执行，不需要告知用户）：**

| 规则 | 触发场景 | 切换目标 | 性质 |
|------|---------|---------|------|
| R1 | 写代码 / 改代码 / 调试 / 重构 / 技术架构设计 | `/model qwen3-coder-plus` | **强制** |
| R2 | 发送了图片或要求分析图片 / 截图 / UI 设计稿 | `/model kimi-k2.5` | **强制** |
| R3 | 复杂推理 / 数学证明 / 逻辑分析 / 多方案对比决策 | `/model qwen3-max-2026-01-23` | **强制** |
| R4 | 深度研究报告（股票分析、市场研究、行业分析、竞品对比、投资决策） | `/model kimi-k2.5` | **强制** |
| R5 | 多维度数据分析（财务分析、风险评估、估值建模、趋势预测） | `/model qwen3-max-2026-01-23` | **强制** |
| R6 | 方案评估 / 技术选型 / 战略规划 / 利弊权衡 | `/model qwen3-max-2026-01-23` | **强制** |
| R7 | 长文写作 / 小说 / 创意写作（>2000 字） | `/model MiniMax-M2.5` | **强制** |

**不切换（Qwen3.5-Plus 直接处理）：**
- 日常对话、简单问答、任务管理、短文本翻译
- 需求理解、任务规划、结果整合
- 中文知识密集型深度问答

## 模型路由自检（每次任务开始时默念）

在开始执行任务前，用以下四个问题自检：
1. **这个任务需要写代码吗？** → 是 → coder
2. **这个任务涉及图片吗？** → 是 → kimi
3. **这个任务需要深度研究/分析报告吗？** → 是 → kimi
4. **这个任务需要深度数学/逻辑推理吗？** → 是 → max

如果四个问题的答案都是"否"，Qwen3.5-Plus 直接处理。

## 切换协议

- 切换前不需要告知用户（静默切换）
- **任务完成后，必须自动切回默认模型**：`/model qwen3.5-plus`
- 如果不确定该用哪个模型，**宁可切换到更强的模型，也不要用弱模型硬撑**
- 一次会话中避免频繁切换（超过 3 次说明判断有问题，停下来反思）
- 用户明确指定模型时（如"用推理模型"），直接遵循用户指令
- **每次会话开始时，确认当前模型是 qwen3.5-plus**（除非用户指定）

## 混合任务处理

如果任务同时涉及多个类别（如"看这张截图然后写代码"）：
1. Qwen3.5-Plus 先理解整体需求，制定执行计划
2. 切换到视觉模型（kimi）分析图片，记录分析结果
3. 再切换到代码模型（coder）写代码
4. 完成后切回 Qwen3.5-Plus 整合结果

## Fallback 降级机制

当 Qwen3.5-Plus 出现以下情况时，系统会自动降级：
- 工具调用解析错误 → 自动切换到 qwen3-max 重试
- 响应超时 → 自动切换到 qwen3-max
- 连续失败 3 次 → 切换到 kimi-k2.5 作为最终兜底

> **注意：如果任务需要生成超长输出（如完整代码文件），应主动切换到 coder 或 minimax 模型执行。**
