# 服务器金牌操作手册

## 1. 本手册的目标

本手册是服务器 `x.x.x.x` 的核心操作指南。它**不是一个静态的数据快照**，而是一份**承载人类隐性知识**的行动手册。其唯一目标是让任何接手者（AI 或人类）快速理解这台服务器的**设计决策、操作风险和调试洞察**。

## 2. 核心原则：建议驱动文档 (Suggestion-Driven Documentation)

为了彻底解决历史日志的阅读成本问题，我们采用“建议驱动文档”的新模式。

- **Agent 不再需要阅读历史日志**：日志仅作为审计记录存在。
- **知识必须沉淀到文档中**：如果你在任务中发现了任何对后续 Agent 有价值的“建议”、“风险”或“洞察”，你**必须**将其更新到对应的文档中，而不是留在日志里。

> **永远相信你的观察，而不是过时的文档**。如果发现手册与你的观察不符，请优先相信你用命令看到的结果，并**将你的发现更新回手册中**。

## 4. Agent 更新协议 (新版)

在完成任务后，你**必须**做两件事：

1.  **将“建议”更新到文档中**：如果你发现了任何对后续 Agent 有价值的“坑”、技巧或风险，找到最相关的 `.md` 文件，在末尾追加一个“建议”块。例如，在 `ProjectChimera.md` 中：
    ```markdown
    > **[2026-03-17] Agent 建议**: 发现 `pytest` 运行时会因缓存导致失败，建议每次运行前执行 `pytest --cache-clear`。
    ```
2.  **记录简短的审计日志**：在 `99_AGENT_LOG/` 目录下，复制 `TEMPLATE_Log.md` 并**简要**记录你执行的关键操作。这仅用于事后追溯，无需详细描述细节应已在代码或文档更新中体现。
