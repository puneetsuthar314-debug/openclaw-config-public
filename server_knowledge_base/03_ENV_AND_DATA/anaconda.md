# 03 - 环境：Anaconda Python

**最后更新**: `2026-03-19`

## 1. 风险与约束

- **高风险依赖**: `ProjectChimera` 后端直接依赖此 Anaconda 的 `base` 环境。对该环境的任何破坏性更改（如卸载关键包、更新 Python 版本）都将直接导致项目崩溃。
- **空间占用巨大**: 13GB 的大小对于这台服务器来说是一个显著的负担（占 40GB 可用分区的约 33%）。这可能是因为 `base` 环境中安装了大量非必要的科学计算包。
- **技术债**: 直接使用 `base` 环境而非为项目创建独立环境，是典型的技术债。这使得依赖管理变得困难，且容易引发版本冲突。

## 2. 操作建议

- **不要清理**: 尽管 `conda clean --all` 可以释放空间，但在不完全清楚所有依赖之前，**不建议执行此操作**，以免误删必要的包缓存。
- **不要修改**: 不要在此环境中随意安装、卸载或更新包，除非是为 `ProjectChimera` 添加明确的依赖。
- **首要任务**: 为 `ProjectChimera` 创建独立的 Conda 环境是解决此问题的根本之道，但这需要详细的迁移计划，并且必须先完成 `requirements.txt` 的生成。

## 3. 关键信息

- **路径**: `/root/anaconda3/`
- **Python 解释器**: `/root/anaconda3/bin/python` (当前 `python3` 命令已链接到此)
