# Agent 操作日志

## 关键操作

1. 采集服务器实际环境数据（cgroups版本、包管理器、内存、systemd版本等）
2. 发现关键约束：cgroups v1（非v2）、可用内存仅1.4GB
3. 对增强版方案中10个方案进行广泛可行性调研（每个方案3个搜索方向）
4. 评估结果：4个方案可执行（需修改）、6个方案放弃
5. 输出修订后的执行路线图

## 关键发现

- cgroups v1 环境下 MemoryHigh/MemoryMax 完全不可用，需用 MemoryLimit
- --single-process 非内存泄漏根因，Chromium headless 长时间运行本身有固有泄漏
- systemctl mask 对 /etc/systemd/system/ 下的文件可能不生效，应移动文件
- earlyoom 比 nohang 更简单轻量，更适合当前环境
- 阿里云云监控已有 agent 在运行，无需额外部署监控平台

## 风险提示

- system-level 服务仍在崩溃循环中（disabled 但 active），需尽快处理
- Chromium 当前占用 5.4GB (70.7%)，随时可能触发 OOM
