# Agent 操作日志: 双搜索共存配置

- **日期**: 2026-03-17
- **任务**: 配置 OpenClaw 双搜索共存（内置 Kimi + MCPorter 阿里云 MCP）
- **触发原因**: 用户希望同时使用阿里云免费搜索和内置 Kimi 搜索

## 关键操作

1. **禁用 DingTalk 插件的 webSearch**
   - `openclaw config set channels.clawdbot-dingtalk.aliyunMcp.tools.webSearch.enabled false`
   
2. **启用内置 Kimi 搜索**
   - `openclaw config set tools.web.search.enabled true`
   - Provider: kimi, API Key: KIMI_API_KEY (已配置在 .env 中)

3. **修补 DingTalk 插件源码**
   - 文件: `/usr/lib/node_modules/clawdbot-dingtalk/dist/index.js`
   - 原因: 插件无条件注册所有工具，不检查 enabled 状态，导致即使配置 webSearch.enabled=false 仍注册 web_search 造成名称冲突
   - 修改: 在工具注册循环前加入 toolEnabledMap 检查，enabled=false 的工具跳过注册
   - 备份: `index.js.bak`

4. **安装 MCPorter 并接入阿里云搜索**
   - `npm install -g mcporter`
   - `mcporter config add WebSearch https://dashscope.aliyuncs.com/api/v1/mcps/WebSearch/mcp --transport http --header 'Authorization=Bearer sk-xxx'`
   - MCPorter 配置文件: `/root/.mcporter/mcporter.json`

## 注意事项

- 插件更新 (npm update clawdbot-dingtalk) 后需重新打补丁
- core-search-sync.js 中的 auto-disable 逻辑在 webSearch.enabled=false 时不会触发，无需额外处理
- MCPorter 的 WebSearch 工具名与内置 web_search 不同，不会冲突
