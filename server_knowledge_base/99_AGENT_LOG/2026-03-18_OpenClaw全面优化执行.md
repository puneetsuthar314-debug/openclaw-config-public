# 2026-03-18 OpenClaw 全面优化执行

## 任务概述
对 OpenClaw 系统进行全面健康诊断、问题验证、方案调研和优化执行。

## 关键操作

### 1. 停止 system-level 崩溃循环
- 将 /etc/systemd/system/openclaw-gateway.service 移动到 /root/backup/
- 执行 systemctl daemon-reload
- 结果：崩溃循环彻底停止（之前每天 2000+ 次崩溃）

### 2. 安装 earlyoom OOM 防护
- dnf install earlyoom (v1.6.2)
- 配置：-m 10 -s 5 --prefer chromium --avoid sshd/openclaw
- systemctl enable earlyoom && systemctl start earlyoom

### 3. 配置 MemoryLimit
- 创建 /root/.config/systemd/user/openclaw-gateway.service.d/override.conf
- MemoryLimit=5G (cgroups v1 兼容指令)

### 4. 修复健康检查脚本
- 端口检测：ss -tlnp → curl http://localhost:18789
- Chromium 内存阈值：500MB → 3000MB
- 新增自动重启逻辑（kill -TERM + openclaw browser start）

### 5. 新增 Chromium 看门狗
- 脚本：/usr/local/bin/openclaw-chromium-watchdog.sh
- cron：每 2 小时执行一次
- 阈值：3GB RSS

### 6. 重启 Chromium
- 手动 kill 旧进程（5.8GB）
- OpenClaw 自动重启新实例（253MB）
- 系统可用内存从 851MB 恢复到 6.4GB

### 7. 清理磁盘
- journal 日志 vacuum 到 200MB
- dnf 缓存清理
- 磁盘使用率从 74% 降到 72%

## 验证结果
- system-level 服务：已不存在 ✓
- earlyoom：active (running) ✓
- MemoryLimit：5G ✓
- OpenClaw user-level：active (running) ✓
- Chromium：253MB ✓
- 端口 18789：正常监听 ✓
- 磁盘：72% ✓
- 可用内存：6.4GB ✓

## 注意事项
- `openclaw browser restart` 命令不存在，需用 kill + `openclaw browser start`
- cron 环境下 ss 命令无法检测 user-level 进程的端口监听
- earlyoom 配置文件位于 /etc/default/earlyoom
